## 1.1.0 - 2026-04-13

### Added

- Prints contextual settings in verbose mode level 2 :
  That will allow to help to find error when configuration files does not exist, and many more other use cases

- `colors` action option to choice between pre-settings declaration of plugin

Pre-settings example:

```json
{
    "plugins": [
        {
            "plugin": "\\Bartlett\\CaptainHookBinPlugin\\BinPlugin",
            "options": {
                "ansi-colors-flag": "--ansi",
                "auto-colors-flag": "--colors=auto",
                "always-colors-flag": "--colors=always",
                "never-colors-flag": "--colors=never"
            }
        }
    ]
}
```

- a generic syntax to add colors flag to all your CLI tools : `\\Bartlett\\CaptainHookBinPlugin\\BinPlugin.colors-flag`

For example:

```json
{
    "actions": [
        {
            "action": [
                "{$ENV|value-of:XDG_BIN_HOME}ecs",
                "{$ENV|value-of:\\Bartlett\\CaptainHookBinPlugin\\BinPlugin.colors-flag|cache:false}"
            ]
        }
    ]
}
```

- a generic syntax to add configuration file to all your CLI tools : `\\Bartlett\\CaptainHookBinPlugin\\BinPlugin.config-file`

For example:

```json
{
    "actions": [
        {
            "action": [
                "{$ENV|value-of:XDG_BIN_HOME}ecs",
                "--config={$ENV|value-of:\\Bartlett\\CaptainHookBinPlugin\\BinPlugin.config-file|cache:false}"
            ]
        }
    ]
}
```

### Changed

- `config-file` action option allow now placeholder replacement syntax

For example:

```json
{
    "options": {
        "config-file": "{$ENV|value-of:XDG_CONFIG_HOME}/ecs.php"
    }
}
```
